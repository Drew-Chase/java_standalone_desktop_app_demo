# Standalone Desktop App Demo

A demo of a self-contained, standalone desktop application built with Java and Swing.
The build produces a native Windows launcher (`.exe`) bundled with a minimal,
custom Java runtime (via `jlink`) so the app runs on machines **without** a
system-wide Java installation.

This project is for fun and educational purposes, not meant for production use.

## What it builds

Running the distribution build assembles everything into `build/dist`:

- **`app_demo.exe`** — a small native C launcher (built with CMake) that
  bootstraps the bundled runtime and starts the app.
- **`bin/app_demo.jar`** — the application jar (Swing "Hello World").
- **`jre/`** — a minimal Java runtime trimmed to just the modules the app needs
  (`java.desktop`), generated by `jlink`.

## Prerequisites

The Java side and the native launcher each have their own toolchain requirements.

### Java / Gradle

- **JDK 21 or later** — must be a full JDK that includes `jlink` (the IntelliJ
  JBR does **not** ship `jlink`). Gradle's toolchain support will locate or
  download a matching JDK; the build pins `JavaLanguageVersion 21`.
- **Gradle** — no separate install needed; use the bundled **Gradle wrapper**
  (`./gradlew`). The wrapper targets Gradle 9.5.

### Native launcher (Windows)

These are required to build the `.exe`. They must be on your `PATH`.

- **CMake 4.2 or later**
- **Ninja** (`ninja.exe` is invoked by the CMake build)
- **A C compiler / toolchain** targeting C23 — e.g. MinGW-w64 (GCC). The
  launcher links statically (`-static-libgcc -static-libstdc++ -static`) and
  builds as a GUI-subsystem executable (no console window).

> The full `assembleDistrib` build is currently **Windows-only** because the
> launcher is a Win32 GUI executable.

## How to build

1. Clone the repository.
2. Ensure the prerequisites above are installed and on your `PATH`.
3. Build the full distribution:

   ```sh
   ./gradlew assembleDistrib --rerun-tasks
   ```

4. The runnable output is in `build/dist` — launch `app_demo.exe`.

### Useful individual tasks

| Task              | Description                                                |
|-------------------|------------------------------------------------------------|
| `jar`             | Builds the application jar with the `Main-Class` manifest. |
| `cmakeConfigure`  | Configures the CMake launcher project.                     |
| `cmakeBuild`      | Compiles the native launcher executable.                   |
| `jlink`           | Builds the minimal Java runtime image into `dist/jre`.     |
| `assembleDistrib` | Runs all of the above and assembles `build/dist`.          |

## Project layout

```
├── build.gradle                       # Gradle build + CMake/jlink/dist tasks
├── launcher/
│   ├── CMakeLists.txt                  # Native launcher build config
│   └── main.c                          # C launcher that bootstraps the JRE
└── src/main/java/com/github/drewchase/
    └── Main.java                       # Swing entry point
```

## Notes

- `assembleDistrib --rerun-tasks` forces a clean rebuild. For incremental
  builds you can drop `--rerun-tasks`; Gradle tracks the C sources and jar
  inputs and rebuilds only what changed.
- If you toggle the launcher between GUI and console subsystem, wipe the CMake
  build directory — with the Ninja (single-config) generator the subsystem is
  decided at configure time and a stale cache will keep the old setting.
- `jlink` refuses to write into an existing directory, so the `jlink` task
  clears `dist/jre` before each run.
